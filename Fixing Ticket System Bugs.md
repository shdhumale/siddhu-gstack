### Root Cause Analysis & Fixes

1.  **Bug 1 (Registration Failure) & Bug 5 (Update Password Failure)**:
    *   **Root Cause**: Two issues were found. First, the backend returned plain text messages (e.g., "User registered successfully!") while the frontend's `HttpClient` expected JSON by default. This caused a parsing error on the frontend side, even if the request was successful on the server. Second, the database credentials in [application.properties](cci:7://file:///c:/All_Antigravity_Project_Workspace/siddhu-gstack/ticketserver/src/main/resources/application.properties:0:0-0:0) (`root`/`root`) did not match the setup in [docker-compose.yml](cci:7://file:///c:/All_Antigravity_Project_Workspace/siddhu-gstack/docker-compose.yml:0:0-0:0) (`dbuser`/`dbpassword`).
    *   **Fix**: Modified the backend [AuthController](cci:2://file:///c:/All_Antigravity_Project_Workspace/siddhu-gstack/ticketserver/src/main/java/com/gstack/ticketserver/controller/AuthController.java:14:0-89:1) to return JSON objects (e.g., `{"message": "..."}`). Updated [application.properties](cci:7://file:///c:/All_Antigravity_Project_Workspace/siddhu-gstack/ticketserver/src/main/resources/application.properties:0:0-0:0) to use the correct database credentials.

2.  **Bug 2 (Dashboard Blank), Bug 3 (Create Ticket Error), Bug 4 (Search Screen Error)**:
    *   **Root Cause**: The [AuthInterceptor](cci:2://file:///c:/All_Antigravity_Project_Workspace/siddhu-gstack/frontend/src/app/core/auth.interceptor.ts:5:0-20:1) was implemented but **not registered** in the Angular bootstrap process. Consequently, the JWT token was never sent with HTTP requests. Since the backend requires authentication for all ticket-related endpoints, the server rejected these calls with a `403 Forbidden` error.
    *   **Fix**: Registered the [AuthInterceptor](cci:2://file:///c:/All_Antigravity_Project_Workspace/siddhu-gstack/frontend/src/app/core/auth.interceptor.ts:5:0-20:1) in [main.ts](cci:7://file:///c:/All_Antigravity_Project_Workspace/siddhu-gstack/frontend/src/main.ts:0:0-0:0) using `withInterceptorsFromDi()` and the `HTTP_INTERCEPTORS` provider.

3.  **Bug 4 (Search screen specific issue)**:
    *   **Root Cause**: The search logic used `.toLowerCase()` on the ticket description. If a ticket was created without a description (null), this caused a runtime error in the Angular application, breaking the search results list.
    *   **Fix**: Added optional chaining (`t.description?.toLowerCase()`) to safely handle null descriptions.
