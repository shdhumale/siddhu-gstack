## Problem Statement
The user needs a browser-based Simple Ticket Management System using Angular, Spring Boot, and MySQL to view, create, update, delete, and search for tickets.

## Demand Evidence
Small teams seeking a "no-fuss" internal tracking tool without the overhead of an authentication system.

## Status Quo
Currently, tracking is either non-existent, done via spreadsheets, or heavy existing solutions that require full authentication and onboarding.

## Target User & Narrowest Wedge
Target user: Small teams that need a rapid internal tracking tool.
Narrowest wedge: A single-dashboard Ticket CRUD app without login overhead.

## Constraints
- No authentication or user roles.
- Single user/local focus out of the box.

## Premises
1. **The Heavy Stack Premise:** Angular + Spring Boot + MySQL is heavy but acceptable for consistency and scalability. (AGREED)
2. **The Open Access Premise:** Anyone on the network can edit. (AGREED)
3. **The Deployment Premise:** Requires easy spin-up (e.g. Docker Compose). (AGREED)

## Approaches Considered
### Approach A: The Enterprise Lite (CHOSEN)
Angular frontend, Spring Boot REST API, MySQL DB. Highly scalable but requires managing 3 pieces.
### Approach B: The Zero-Config Backend
Embedded SQLite/H2 instead of MySQL.
### Approach C: The Monolith
Single runnable JAR or alternative fullstack framework.

## Recommended Approach
**Approach A:** The Enterprise Lite, coupled with a `docker-compose.yml` to simplify spinning up the 3 services for small teams.

## Open Questions
- Are there any specific search filters required beyond basic keywords?

## Success Criteria
- User can view a dashboard of all tickets.
- User can create a new ticket with New, Assigned, Done, or Escalate status.
- User can update and delete tickets.
- System relies on Angular for UI, Spring Boot for API, MySQL for storage.

## Distribution Plan
Repository containing the Angular source, Spring Boot source, and a unified `docker-compose.yml` that will build and run the Angular web server, the Spring Boot application container, and the MySQL database container simultaneously.

## Dependencies
- Docker & Docker Compose for the seamless distribution plan.
- Java 21+ for Spring Boot.
- Node.js for Angular.

## The Assignment
Initialize the Git repository, scaffold the Angular frontend and the Spring Boot backend, and build the initial `docker-compose.yml` file to run MySQL.
