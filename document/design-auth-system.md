## Problem Statement
The user needs a secure and aesthetically pleasing authentication system for the Ticket Management System, including login, registration, and password update functionalities with client-side validation.

## Target User
Internal team members or individual users who need to secure their ticket data.

## Implementation Details

### Frontend (Angular)
- **LoginComponent**: Uses Reactive Forms for email (pattern-validated) and password (required). Styled with glassmorphism.
- **RegisterComponent**: Same validation as login, with an additional minimum length check for passwords.
- **UpdatePasswordComponent**: Allows users to change their password by verifying their current password and confirming the new one.
- **AuthService**: Handles HTTP calls to the Spring Boot backend and manages the JWT token in `localStorage`.
- **Navigation**: Conditionally shows the dashboard navigation only for authenticated users.

### Backend (Spring Boot)
- **AuthController**: 
  - `POST /auth/login`: Authenticates user and returns JWT.
  - `POST /auth/register`: Creates a new user record with hashed password.
  - `POST /auth/update-password`: Verifies old password and updates to new one.
